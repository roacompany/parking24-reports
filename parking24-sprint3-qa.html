🚨 진행 중 발견 + 해결

B7-2 audit-logs GET → 403

# PR #141 머지 후 E2E 1차 실행 결과
GET /api/admin/audit-logs?since=24h&limit=10
status: 403   ← SUPER_ADMIN 인데도 fail

Root cause

requireApiAuth('audit-logs', 'read') 가 hasPermission 호출. hasPermission 은 ADMIN_PERMISSIONS[resource] 조회 → 'audit-logs' 키가 map 에 없어 undefined → fail → 403.

Hotfix #142

src/lib/constants.ts ADMIN_PERMISSIONS 에 'audit-logs' 키 추가:

"audit-logs": { SUPER_ADMIN: "read", LOT_ADMIN: "none", PARTNER_ADMIN: "none", PARTNER_SUPER_ADMIN: "none" }

middleware ACL 과 동일 정책. 머지 + Vercel 배포 후 재검증 통과.

학습

새 admin endpoint 추가 시 middleware PERMISSIONS map + ADMIN_PERMISSIONS map 양쪽 모두 추가 필요. 향후 lint/test로 drift 자동 검출 검토.
