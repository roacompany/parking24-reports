A2 · Migration 031 인덱스 PASS

1. pg_indexes — 3개 모두 production 생성

$ SELECT indexname, indexdef FROM pg_indexes
  WHERE tablename = 'reservations' AND indexname LIKE 'idx_%created%';

idx_reservations_created_at
  USING btree (created_at DESC)

idx_reservations_partner_created  ← partial
  USING btree (partner_id, created_at DESC)
  WHERE (partner_id IS NOT NULL)

idx_reservations_airport_created
  USING btree (airport_id, created_at DESC)

2. EXPLAIN ANALYZE (현재 39 row, planner 자동 선택)

EXPLAIN ANALYZE SELECT id, created_at FROM reservations
ORDER BY created_at DESC LIMIT 50;

Limit  (cost=3.42..3.52)
  -> Sort
       -> Seq Scan on reservations  (Buffers: hit=2)
Execution Time: 0.205 ms
참고: 현재 reservations 39 row 라 PostgreSQL planner 가 Seq Scan 을 더 싸게 판단해서 인덱스 미사용. 이는 정상 동작 — 데이터 누적되면(수천 row 이상) planner 가 자동으로 인덱스로 전환.

3. 인덱스 적중 가능성 검증 (SET enable_seqscan = off)

SET LOCAL enable_seqscan = off;
EXPLAIN ANALYZE SELECT ... ORDER BY created_at DESC LIMIT 50;

Limit  (cost=0.14..5.03)
  -> Index Scan using idx_reservations_created_at
Execution Time: 1.399 ms  ← 인덱스 실제 사용 가능 ✓

인덱스가 실제 사용 가능 함을 확인. 광고 시작 후 데이터 누적되면 planner 가 자동 전환.
