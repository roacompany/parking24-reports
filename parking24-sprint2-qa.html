Root Cause

PR #132에서 src/app/api/admin/route.ts 에 unstable_cache tag 상수를 export:

export const ADMIN_DASHBOARD_CACHE_TAG = "admin-dashboard";

Next.js 15 Route Handler 는 HTTP methods + runtime/dynamic/revalidate 등 specific export 만 허용. 임의 상수 export 금지 → build fail. PR #132 build 실패 → main이 PR #132 코드를 포함하므로 이후 모든 PR (#133/#134/#135) 도 같은 main 위에서 build 시도 → 동일 실패.

왜 CI 에서 안 잡혔나

기존 verify:ci = typecheck + lint + arch + test — next build 누락. typecheck/lint 는 임의 export OK, 빌드만 fail.

Hotfix (PR #137)

  1. src/lib/admin-cache-tags.ts 신규 — 상수 격리
  2. route.ts 에서 import 로 변경
  3. package.json verify:ci 에 npm run build 추가 (CI gap fix)

머지 후 Vercel build 성공 → PR #132~#135 변경 자동 prod 반영.

운영 도구 (PR #136)

.github/workflows/vercel-promote.yml — workflow_dispatch 로 임의 deployment 강제 promote 가능. 향후 유사 상황 즉시 대응.
